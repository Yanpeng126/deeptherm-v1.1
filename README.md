# DeepTherm Reproduction

An independent reproduction of **DeepTherm** (Wang et al., *Energy and AI* 24, 2026, [doi:10.1016/j.egyai.2026.100677](https://doi.org/10.1016/j.egyai.2026.100677)), a unified deep learning model for predicting thermochemistry properties of gas-phase molecules.

This implementation is rewritten from the paper text on top of [chemprop v2.2.3](https://github.com/chemprop/chemprop). It is not affiliated with the original authors. The original code is at https://github.com/tairan-w/DeepTherm.

## Scope

| Paper component | This repository |
| --- | --- |
| DMPNN bond-centered message passing (Section 2.1) | `src/model.py` / `BondAttentionEncoder.mp` |
| Bond-level multi-head global attention (Section 2.1) | `src/model.py` / `BondAttentionEncoder.attn` |
| Attention-weighted bond-to-atom aggregation (Eq. for h^att_i) | `src/model.py` / `scatter_reduce` on `edge_index[1]` |
| Graph readout f_agg (NormAggregation) | `src/model.py` |
| Feed-forward predictor | `src/model.py` / `RegressionFFN` |
| ECFP 1024-bit concatenation (Section 2.6) | `src/model.py` / `ECFPProjection` |
| Five extra RDKit atom/bond features (SI Fig. 3) | `src/dataset.py` / `extra_features` (implemented, disabled by default) |
| 10-model ensemble with inverse-MAE weighting (Section 2.3) | `src/ensemble.py`, `run_ensemble.sh` |
| Random 81:9:10 splits | `src/dataset.py` / `split_with_fixed_test` |
| Hierarchical complexity-based 9:1 splits | `src/dataset.py` / `split_by_complexity` |
| QM9 transfer learning (Section 2.2) | **Not implemented**; the codebase exposes a slot for loading pretrained encoder weights once a checkpoint is available |

## Results

10-model ensemble test MAE on the 4,997-species curated dataset.

**Random splits** (compare against Fig. 6a-c of the paper):

| Target | This work | Paper (full DeepTherm) |
| --- | --- | --- |
| ΔHf,298K (kcal/mol) | 2.38 | 1.25 |
| S298K (cal/mol/K) | 1.49 | 1.31 |
| Cp,300K (cal/mol/K) | 0.76 | 0.86 |
| Cp,400K (cal/mol/K) | 0.78 | 0.82 |
| Cp,500K (cal/mol/K) | 0.75 | 0.87 |
| Cp,600K (cal/mol/K) | 0.72 | 0.84 |
| Cp,800K (cal/mol/K) | 0.70 | 0.92 |
| Cp,1000K (cal/mol/K) | 0.69 | 0.84 |
| Cp,1500K (cal/mol/K) | 0.63 | 0.82 |

**Hierarchical complexity-based splits** (compare against Fig. 6d-f):

| Target | This work | Paper (full DeepTherm) |
| --- | --- | --- |
| ΔHf,298K (kcal/mol) | 3.94 | ~2.1 |
| S298K (cal/mol/K) | 3.24 | ~2.4 |
| Cp,300K (cal/mol/K) | 1.94 | ~1.9 |

Eight of nine random-split targets reach or exceed the paper. The residual ΔHf gap reflects the absence of QM9 transfer learning together with implementation details that the paper does not fully specify.

## Installation

```bash
git clone https://github.com/<your-handle>/deeptherm-reproduction.git
cd deeptherm-reproduction
conda create -n deeptherm python=3.11 -y
conda activate deeptherm
pip install chemprop==2.2.3 rdkit pandas numpy lightning
```

A100 or any single GPU with >=24 GB memory is recommended for full ensemble runs.

## Data preparation

Place the supplementary curated dataset (Supplementary Data 1 from the paper) as `data/raw/curated_dataset.xlsx`, then:

```bash
python src/prepare_data.py
```

This writes a canonicalized CSV to `data/curated_4997.csv` containing 4,997 species with 9 thermochemistry targets.

## Training a single model

```bash
python src/train.py \
    --csv data/curated_4997.csv \
    --save-dir runs/single \
    --epochs 300 \
    --batch-size 32 \
    --d-hidden 600 \
    --depth 5 \
    --ecfp-bits 1024 \
    --seed 42
```

Per-target test MAE is printed to stdout and a `predictions.npz` is written to the save directory.

## Running the 10-model ensemble

Hyperparameters in `run_ensemble.sh` are set to the values that worked best in our grid search.

**Random split**:
```bash
SPLIT_MODE=random OUT_ROOT=runs/ensemble_random bash run_ensemble.sh
```

**Hierarchical complexity-based split**:
```bash
SPLIT_MODE=complexity OUT_ROOT=runs/ensemble_complexity bash run_ensemble.sh
```

Each ensemble takes 2.5-3.5 hours on a single A100. The aggregated predictions and per-target MAE/RMSE are printed at the end.

## Differences from the original code

The published code at https://github.com/tairan-w/DeepTherm does not run as is, and even with environment downgrades it does not train the architecture described in the paper. This repository is rewritten from the paper text, so the code structure does not follow the original.
